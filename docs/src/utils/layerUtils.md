# `utils/layerUtils.ts`

## 📋 Purpose

Tree-manipulation utilities for the recursive layer structure. All functions treat the layer array as an **immutable tree** and return new arrays/objects rather than mutating in place.

## 🔧 Functions

---

<code>findLayerById(layers: Layer[], id: string): Layer | undefined</code>

Recursively searches the layer tree (depth-first) for a layer whose `id` matches the given string. Returns the `Layer` object if found, or `undefined` if the id does not exist anywhere in the tree (including nested `children`).

---

<code>findParentNode(layers: Layer[], id: string): Layer | null</code>

Recursively walks the layer tree to find the **direct parent** of the layer with the given `id`. Checks each group's `children` array; if a child's `id` matches, the current group is returned. Returns `null` if the layer is a root-level layer (has no parent group).

---

<code>removeNode(layers: Layer[], id: string): Layer[]</code>

Returns a **new layer array** with the layer identified by `id` removed, at any depth of the tree. Filters out matching root-level layers and recursively removes from `children` of group layers. Does not mutate the original array.

---

<code>insertNode(layers: Layer[], node: Layer, parentId?: string | null): Layer[]</code>

Inserts a `node` into the layer tree. If `parentId` is `null` or `undefined`, the node is prepended to the root-level array. Otherwise, the tree is walked recursively until the matching parent group is found, and the node is prepended to that group's `children`. Returns a new array without mutation.

---

<code>updateNode(layers: Layer[], id: string, updates: Partial<Layer>): Layer[]</code>

Returns a new layer tree with the target layer shallow-merged with `updates` (`{ ...layer, ...updates }`). Recursively updates inside group `children` as needed. Does not mutate the original tree.

---

<code>flattenTree(layers: Layer[]): Layer[]</code>

Performs a **depth-first flattening** of the nested layer tree into a single flat array. Every group layer is included in the result followed immediately by its descendants. Useful for iterating over all layers regardless of nesting depth.

---

<code>getVisibleRenderOrder(layers: Layer[]): Layer[]</code>

Filters the **top-level** layer array to only include layers where `visible === true`. Returns them in their original order (index 0 = topmost). Note: this currently operates on root-level layers only; nested children should be handled by the rendering component itself.

---

<code>moveNode(layers: Layer[], id: string, direction: 'up' | 'down'): Layer[]</code>

Moves a layer one position **up** or **down** within its sibling list. Works at any depth — first finds the parent (or uses the root array if the layer is top-level), deep-copies the tree, then swaps the element at the found index with its neighbour. Returns the updated tree.

---

<code>reorderNodes(layers: Layer[], draggedId: string, targetId: string, position: 'before' | 'after' | 'inside'): Layer[]</code>

Implements **drag-and-drop reordering** in the layer panel. Removes the dragged node from the tree, then re-inserts it relative to `targetId`:
- `'inside'` → becomes a child of `targetId`'s group.
- `'before'` / `'after'` → inserted as a sibling immediately before or after `targetId` at any nesting depth.

Returns a new, fully-reordered layer tree.

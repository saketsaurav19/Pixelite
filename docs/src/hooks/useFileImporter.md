## 🔧 Functions

---

<code>useFileImporter(): { handleFileImport }</code>

React hook that connects to the Zustand store and exposes a single `handleFileImport` function. Internally reads store actions (`setLayers`, `setDocumentSize`, `recordHistory`, etc.) so that callers only need to provide a `File` object and an optional `isPlace` flag.

Returns: `{ handleFileImport }` — an object containing the handler function below.

---

<code>handleFileImport(file: File, isPlace: boolean = false): Promise&lt;void&gt;</code>

Dispatches file import to the `ImportEngine` and creates layers based on the detected file type:

- **PSD** — parses via `workerExportBridge.parsePSD`, flattens the PSD layer tree into individual `image` layers, preserving names, positions, opacity, blend modes and visibility.
- **GIF** — maps each animation frame to a separate `image` layer, positioned at the document centre when placing.
- **SVG** — uses the ImportEngine's pre-built layer array, reversing order so the topmost SVG element renders on top.
- **PDF** — imports each page as a group layer tree. When placing into an existing document, each page group is offset to centre within the current canvas.
- **Standard image (PNG / JPEG / WEBP / etc.)** — creates a single `image` layer. On first open, resizes the document to match the image dimensions; when placing into an existing document, centres the image.

When `isPlace = false`, resets the project ID and history before loading. When `isPlace = true`, appends new layers to the existing layer stack without touching history or document size (except for the image format which still centres).
